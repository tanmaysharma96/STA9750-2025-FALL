Step 1: Data Acquisition:

For the district boundary data, I manually downloaded the official City Council (Clipped to Shoreline) shapefile from the NYC Department of Planning website and saved the ZIP file into my data/mp03/ directory. My R function then checks for the existing ZIP file, unzips it only if needed, reads the shapefile using sf::st_read(), and converts the geometry to WGS84 to ensure alignment with the tree dataset. This avoids repeated downloads and provides a clean, reproducible workflow for loading the district boundaries.

get_nyc_council_districts <- function(
  dest_dir = "data/mp03",
  zip_name = "council_districts.zip"
) {
  # Path to zip file (already provided locally)
  zip_path <- file.path(dest_dir, zip_name)

  if (!file.exists(zip_path)) {
    stop("ZIP file not found. Place it in data/mp03/ as council_districts.zip")
  }

  # Unzip only if needed
  shp_files <- list.files(dest_dir, pattern = "\\.shp$", full.names = TRUE)
  if (length(shp_files) == 0) {
    unzip(zip_path, exdir = dest_dir)
    shp_files <- list.files(dest_dir, pattern = "\\.shp$", full.names = TRUE)
  }

  if (length(shp_files) == 0) {
    stop("No .shp file found after unzipping. Check ZIP contents.")
  }

  # Read shapefile
  dist_sf <- sf::st_read(shp_files[1], quiet = TRUE)

  # Transform to WGS84 for mapping
  sf::st_transform(dist_sf, crs = "WGS84")
}

NYC Tree Points

To obtain the full NYC Street Tree dataset, I used the NYC OpenData SODA2 API, which returns data in paged batches. Because a single request only retrieves the first 1,000 rows, my code iteratively adjusts the $limit and $offset parameters to ‚Äúpage through‚Äù the entire dataset and save each chunk locally. This approach ensures responsible API usage while allowing me to download the complete set of tree points for later spatial analysis.

get_nyc_tree_points <- function(
  base_url = "https://data.cityofnewyork.us/resource/hn5i-inap.geojson",
  dest_dir = "data/mp03/trees",
  limit = 5000   # You can change this if you want smaller/larger chunks
) {
  # Create directory if needed
  if (!dir.exists(dest_dir)) dir.create(dest_dir, recursive = TRUE)

  offset <- 0
  all_files <- c()
  batch_num <- 1

  repeat {
    message("Downloading batch ", batch_num, " with offset = ", offset)

    # Build paged API request
    req <- request(base_url) |>
      req_url_query(`$limit` = limit, `$offset` = offset)

    # Perform request
    resp <- req_perform(req)

    # Parse JSON as text then convert to R list
    content <- resp_body_string(resp)
    json_data <- fromJSON(content)

    # Save raw JSON to file
    file_path <- file.path(dest_dir, paste0("trees_batch_", batch_num, ".geojson"))
    writeLines(content, file_path)
    all_files <- c(all_files, file_path)

    # Number of rows returned
    n_returned <- length(json_data$tree_id)

    # Stop if last batch (< limit means we're at the end)
    if (n_returned < limit) break

    # Otherwise continue to next batch
    offset <- offset + limit
    batch_num <- batch_num + 1
  }

  message("Download complete. Reading and combining all batches...")

  # Read all GeoJSON batches into sf objects and combine
  tree_sf <- map_df(all_files, ~ st_read(.x, quiet = TRUE))

  # Return final sf object
  tree_sf
}

Step 2: R Code for Task 2 (uses httr2, paging, caching, st_read, bind_rows)

library(httr2)
library(sf)
library(dplyr)

get_nyc_tree_points <- function(
  base_url   = "https://data.cityofnewyork.us/resource/hn5i-inap.geojson",
  dest_dir   = "data/mp03",
  limit      = 2000,
  max_batches = Inf
) {
  if (!dir.exists(dest_dir)) {
    dir.create(dest_dir, recursive = TRUE)
  }

  offset     <- 0
  batch_idx  <- 1
  trees_list <- list()

  repeat {
    if (batch_idx > max_batches) {
      message("Reached max_batches = ", max_batches, ". Stopping early.")
      break
    }

    file_path <- file.path(
      dest_dir,
      sprintf("tree_points_%03d.geojson", batch_idx)
    )

    # Download only if this batch file doesn't already exist
    if (!file.exists(file_path)) {
      message("Downloading batch ", batch_idx,
              " (offset = ", offset, ", limit = ", limit, ")")

      req <- request(base_url) |>
        req_url_query(`$limit` = limit, `$offset` = offset)

      resp <- tryCatch(
        req_perform(req),
        error = function(e) {
          warning("Request failed for batch ", batch_idx, ": ",
                  conditionMessage(e))
          return(NULL)
        }
      )

      if (is.null(resp)) {
        message("Stopping because of repeated request errors.")
        break
      }

      writeBin(resp_body_raw(resp), file_path)
    } else {
      message("Using cached file: ", basename(file_path))
    }

    # Read this batch as sf from GeoJSON
    trees_batch <- sf::st_read(file_path, quiet = TRUE)

    # üîß Normalize ALL datetime columns to character so bind_rows won't choke
    dt_cols <- vapply(trees_batch, inherits, logical(1), what = "POSIXt")
    if (any(dt_cols)) {
      trees_batch[, dt_cols] <- lapply(trees_batch[, dt_cols, drop = FALSE], as.character)
    }

    trees_list[[batch_idx]] <- trees_batch

    n_returned <- nrow(trees_batch)

    if (n_returned == 0 || n_returned < limit) {
      message("Final batch size = ", n_returned, ". Finished paging.")
      break
    }

    offset    <- offset + limit
    batch_idx <- batch_idx + 1
  }

  # Final safety pass: make sure any leftover POSIX columns are characters
  trees_list <- lapply(trees_list, function(x) {
    dt_cols <- vapply(x, inherits, logical(1), what = "POSIXt")
    if (any(dt_cols)) {
      x[, dt_cols] <- lapply(x[, dt_cols, drop = FALSE], as.character)
    }
    x
  })

  dplyr::bind_rows(trees_list)
}

NYC Parks Proposal: District 51 Tree Revitalization Program

As a staffer for the NYC Council Member representing District 51, I am proposing a targeted street-tree revitalization program to restore and strengthen the urban canopy across the South Shore of Staten Island. This district has the largest total number of street trees in New York City (91,335 trees), yet a significant portion are in poor, critical, or dead condition, particularly along key residential corridors such as Hylan Boulevard and Amboy Road. The goal of this program is to replace hazardous trees, replant lost canopy, and improve overall environmental resilience.

Proposed Scope of Work

  • Replace 2,000 dead or critically rated trees
  • Plant 1,500 new trees in areas with sparse canopy
  • Remove 1,000 existing stumps to create space for future plantings
  • Perform risk-mitigation work (pruning or treatment) on 500 high-risk trees identified in the NYC Parks dataset

A spatial analysis of NYC OpenData shows that District 51 leads the city in total tree count but also ranks among the higher districts in dead-tree burden. When compared with District 39 (Brooklyn), District 32 (Queens), and District 19 (Queens), District 51 shows both a greater number of failing trees and neighborhoods where canopy loss is accelerating, particularly in coastal zones vulnerable to storm surge, salt exposure, and severe weather events. These quantitative differences illustrate that additional NYC Parks resources allocated to District 51 will have an outsized impact on public safety and long-term canopy health.

A zoomed-in map of District 51 highlights the distribution of tree conditions, with clusters of dead and critical trees visible around Great Kills, Annadale, and Tottenville. A comparative bar chart of dead-tree counts across districts further demonstrates the relative need for targeted intervention in this district.

By directing a focused revitalization effort in District 51, NYC Parks can achieve significant improvements in environmental resilience, pedestrian comfort, and neighborhood aesthetics. This proposal presents a clear, data-driven rationale for prioritizing the South Shore in the next round of discretionary urban forestry funding.
